What is marabunta?

From its definition, marabunta is one of the names given to army ants. The idea of the project is to create something similar using a client-server topology, thinking about node/client as the ant and the server as the nest.

Is a try to apply nature knowledge and patterns into existing networks so on top of the existing tools this could be an option to do extra tasks of simple have a basic communication channel between server/client or the entire network “colony”.

First steps are to create a “colony” having a nest (server), and some clients ants, implementing and finding protocols that could help communication between all nodes in a light and consistent way, in order to send/broadcasts commands from the server to all or a specific node.

Further uses just to mention some could be:

  • monitoring of the nodes itself
  • service discovery
  • backdoor for managing full node not requiring ssh
  • orchestrate distributed tasks
  • remote execution
